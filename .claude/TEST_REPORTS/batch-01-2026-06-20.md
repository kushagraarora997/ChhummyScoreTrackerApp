## Critical Findings

### BUG-01 — CRITICAL: Round flow completely broken for new players
**Severity:** P0 — App is unplayable end-to-end for new players.

**Root cause (confirmed):**
1. `PlayerSetup.tsx` calls `db.players.add(player)` — writes to IndexedDB
2. `PlayerSetup.tsx` updates its own local React state (`available`) — but does NOT update Zustand store
3. `newSession(playerIds)` in `useAppStore.ts:85` saves session + playerIds but never reloads `store.players` from DB
4. `store.players` remains as it was at `init()` time — empty on first run
5. `LiveGame.tsx` builds player display from `store.players` → falls back to `{ name: "Player" }` for all
6. `Overlays()` component filters `store.players` to build "Who Closed?" button list → empty array → zero buttons rendered

**Fix (surgical, 1 line):**
In `src/store/useAppStore.ts`, function `newSession()` (~line 85):
```ts
async newSession(playerIds) {
  const session: Session = { ... };
  await db.sessions.add(session);
+ const players = await db.players.toArray(); // reload fresh from DB
  set({
+   players,
    activeSession: session,
    rounds: [],
  });
},
```
